新月配列
Shingetsu Layout

月配列2-263をベースに、スマホフリック入力の発想を取り入れた
最小かつ最高効率のPCキーボード用かな配列

今すぐ導入する GitHub で見る

特徴

なぜ新月配列なのか

🌙

月配列2-263ベース

実績ある月配列の前置シフト方式を継承。既存の月配列ユーザーも違和感なく移行できます。

📱

1キー統合入力

スマホフリック入力のアイデアを応用。濁音・半濁音・小文字を1キーで入力可能。

⚡

最小打鍵数

高頻度文字は1打鍵、中頻度は2打鍵で入力。長文入力での疲労を大幅に軽減します。

🖥️

マルチプラットフォーム

macOS (Karabiner Elements)・Windows/Linux (hazkey) に対応。主要環境をカバー。

配列構造

3層の前置シフト方式で全かなをカバー

レイヤー 発動方法 用途 打鍵数
Layer 0 そのまま打鍵 高頻度かな文字 1打鍵
Layer 1 ☆ or ★ キー → 文字キー 中頻度かな文字 2打鍵
Layer 2 ☆ → 濁音/半濁音+小文字キー → 文字キー 低頻度かな文字・濁音・半濁音・小文字 3打鍵

導入方法

お使いの環境に合わせてインストール

🍎 macOS — Karabiner Elements

  1. Karabiner Elements をインストール
  2. shingetsu-karabiner-qwerty.json をダウンロード
  3. ファイルを開き、Karabiner Elements の設定にインポート
  4. 「Complex Modifications」で新月配列を有効化

🪟 Windows / 🐧 Linux — hazkey

  1. hazkey をインストール
  2. キーボード配列に合わせてTSVを選択:
    • QWERTY配列用
    • Colemak配列用
  3. hazkey の設定でTSVファイルをインポート

📊 keyboard_analyzer

  1. keyboard_analyzer を開く
  2. shingetsu_analyzer.json をダウンロード
  3. keyboard_analyzer でJSONファイルを読み込み
  4. 打鍵効率を可視化・評価

ダウンロード

必要なファイルを直接ダウンロード

shingetsu-karabiner-qwerty.json Karabiner Elements 用設定ファイル（macOS） shingetsu-ansi-qwerty.tsv hazkey 用ローマ字テーブル（QWERTY配列） shingetsu-ansi-colemak.tsv hazkey 用ローマ字テーブル（Colemak配列） shingetsu_analyzer.json keyboard_analyzer 用配列データ shingetsu-layout.svg 配列図（SVG）
